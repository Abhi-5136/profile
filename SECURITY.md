# Security Policy

## Supported Versions

We actively maintain and update the portfolio website. Security updates and fixes will be applied to the latest version of the site. Older versions or forks may not receive updates.

| Version | Supported          |
| ------- | ------------------ |
| HTML 5  | :white_check_mark: |
| CSS 3   | :white_check_mark: |


## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability, please help us by reporting it responsibly.

### How to report:  
Open a private issue or contact the repository owner directly via email (listed in the GitHub profile).

### What to include:

A clear description of the vulnerability

Steps to reproduce the issue

Potential impact

Suggested fix (if possible)

We will acknowledge receipt within 48 hours and aim to provide a resolution or mitigation plan within 7 days.

## Security Practices
To protect this portfolio website and its visitors, we follow these practices:

Use HTTPS for all connections (via GitHub Pages).

Keep dependencies updated to the latest stable versions.

Avoid storing sensitive or personal data in the repository.

Apply the principle of least privilege for any integrations or APIs.

Regularly review and remove unused code, packages, or secrets.

## Responsible Disclosure
Please do not publicly disclose vulnerabilities until they have been addressed. Coordinated disclosure helps keep everyone safe.
